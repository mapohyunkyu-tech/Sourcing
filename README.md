# MarketScout 시즌 AI v3.4

## 핵심
- 오늘 등록할 품목
- 오늘 진입 품목
- 현재 판매 중 품목과 남은 판매일
- 지금 준비할 품목과 진입까지 남은 일수
- 30일 내 준비 시작 품목
- 21일 내 피크 품목
- 전체 시즌 일정
- 기존 v3 대량수집·이어받기·DB 백업/복원 유지

## 배포
압축을 풀어 GitHub 저장소의 파일을 전부 교체한 뒤 Streamlit Cloud에서 Reboot app을 실행합니다.
기존 DB는 DB 백업 탭에서 복원하면 저장된 분석 결과가 대시보드에 즉시 표시됩니다.

## v3.5 새 데이터로 다시 받기

`🚚 대량 이어받기` 탭의 `🔄 새 데이터로 다시 받기`에서 다음 작업을 할 수 있습니다.

- 특정 품목 다시 받기: 품목명을 줄바꿈 또는 쉼표로 입력
- 카테고리 전체 다시 받기: 대표품목만 또는 모든 세부품목 선택
- 실패 품목 다시 대기: 기존 버튼 유지
- 전체 처음부터 다시 받기: 안전 백업 생성 후 확인문구 `전체초기화` 입력

재수집 대상으로 지정된 품목은 기존 `trend_cache` 원자료와 `analysis_results` 분석값이 삭제되고 `collection_queue`가 `pending`으로 변경됩니다. 품목 마스터와 별칭 정보는 삭제되지 않습니다.
